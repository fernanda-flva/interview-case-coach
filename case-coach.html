Phase 1 — Reframe the Brief
🔑 Before touching any data, challenge what the prompt is actually asking. Strong candidates spend 60–90 seconds here.
Phase 2 — Decompose the Problem
🔑 Break the top-level metric or problem into its component parts. Never diagnose the aggregate — find which component is failing.

By user segment
By geography

Phase 3 — Build Hypotheses
🔑 For each component you've identified, generate a ranked list of falsifiable hypotheses. Each must be testable with the data you have.
Demand increase Supply / capacity drop Discovery failure Value failure Segment fit mismatch Process / design fault External event
Phase 4 — Data Strategy
🔑 For each hypothesis, name the specific metric or query that confirms or kills it. Don't just collect data — decide what each data point proves.
Phase 5 — Check-in Moment
🔑 Check in with the interviewer 2–3 times per case. This is moment 2 (after hypotheses). Show confidence, invite course-correction, protect from wrong paths.
Phase 6 — Recommendation
🔑 End with a decision, not just analysis. Tier your recommendations by speed of implementation. Name what you'd monitor going forward.
